# 0.5.2

Release date: 2026-05-03

## Cloud sink

- New `CloudSink` streams events to a remote ingest endpoint in batches (#135). Configured under `[cloud]` in `mcpr.toml`:
  - `cloud_endpoint`, `cloud_token`, `cloud_server`, `cloud_batch_size`, `cloud_flush_interval_ms`.
- The CLI emits a periodic `HeartbeatEvent` so dashboards stay live even when there is no traffic.

## Event model

- Single `ProxyEvent` per request now carries request, response, and latency in one record. Sinks (sqlite, stderr, cloud) all write one row per request instead of separate request and response rows.
- Wire types reorganized around `LoggedRequest` / `LoggedResponse` (slim logging projection). Raw HTTP body bytes are no longer propagated to sinks; the MCP JSON-RPC payload is still recorded in full.
