# 0.5.41

Release date: 2026-05-04

## Wire envelope

- Each transaction emits one `kind:"request"` envelope instead of separate `request` and `response` envelopes. The payload carries `rpc`, `result`, `http_status`, `latency_us`, and `upstream_us` together. Sinks see one row per transaction; cloud projections no longer have to join the two halves on `request_id`.
- MCP envelopes carry `session_id`. The proxy reads it from the request headers, falling back to the response headers so the `initialize` transaction (where the server announces the session id) is captured.
- HTTP envelopes split the legacy `body_size` field into `request_size` and `response_size`.
- McpBatch flattens to N envelopes, pairing each rpc with the result whose JSON-RPC id matches. Results without a matching rpc are dropped.

## Stderr sink

- Every emitted line includes `ts` (epoch seconds). Transaction lines also include `session_id`, pulled from request headers with a response-header fallback for `initialize`.

## Event model

- `LoggedResponse` slims resource payloads in the logging projection. `resources/list` and `resources/templates/list` keep only `uri`/`name` (and `uriTemplate`) per item; `resources/read` keeps only `uri` per content. Widget HTML and blob bodies no longer reach sinks. The response returned to the client is unchanged.
