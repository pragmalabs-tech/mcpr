# 0.5.0

Release date: 2026-05-01

## Proxy rewrite (proxyV2)

The proxy is rebuilt around a stage pipeline: router, request, response, CSP rewrite, schema tracking, session tracking.

- HTTP transport switched from `reqwest` to `hyper`, with a per-upstream connection pool.
- HTTP envelope is carried through the MCP path so headers and status are preserved end-to-end.
- `text/event-stream` responses are streamed end-to-end via `RouterOutput`.
- `GET` SSE is supported for upstreams that open the stream on GET.
- Relay mode is reinstated on top of the new pipeline.
- Proxy config moved into the proxy layer (`proxy2/proxy_config.rs`).
- Default proxy port is now `3004`.

## Storage and events

- New SQLite sink replaces the previous storage layout; writer and query layers were rewritten against the new schema.
- Schema events are tracked through the proxy pipeline (tool, prompt, resource, resource template).
- Sessions are tracked with optimistic insert and explicit termination on `DELETE`.
- Server info and client info are captured from the `initialize` exchange.

## Removals

- `mcpr observe` command removed.
- Old proxy logic removed; proxyV2 is the only path.
- Schema manager removed; capture is limited to tool, prompt, resource, and resource template.
- Outdated docs removed.

## Fixes

- Accept `null` JSON-RPC request IDs.
- Use the request context method when applying CSP rewrites.

## Other

- Dockerfile updated for the new build layout.
