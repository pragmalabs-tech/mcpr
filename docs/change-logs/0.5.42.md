# 0.5.42

Release date: 2026-05-04

## Wire envelope

- `upstream_us` now reports time to first SSE frame, not time to upstream headers. The `Router` timer span ends when upstream returns response headers; for `text/event-stream` responses the body still streams in afterwards. Previously the gap between headers and the first frame counted as proxy overhead in any `proxy = total - upstream` math, even though the proxy was idle on a socket read. A new `UpstreamBody` span covers that gap and closes when the first frame is yielded; `upstream_us` is the sum of both spans.
- JSON (non-SSE) responses are unchanged: `Router` already covers head + body collection, and no `UpstreamBody` span is opened.

## Timer

- The per-request timer (visible with `MCPR_DEBUG=1`) records a new `UpstreamBody` line for SSE responses. Example dump for a `tools/call` that takes 48ms end to end:

  ```
  [mcpr] timer:
    Parse: 23.79µs
    Router: 8.32ms
    UpstreamBody: 39.71ms
    Encode: 5.14µs
  ```

  Before this release, the same request showed only `Router: 8.32ms` and `Encode: 5.14µs`, leaving the 39ms unaccounted for.
